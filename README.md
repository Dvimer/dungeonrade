# GameTwo

Match-swipe roguelike. Целевая платформа — **Yandex Games (HTML5)**.
Движок: **Godot 4.6** (Compatibility renderer для веб-сборки).

## Запуск

1. Установить Godot **4.6.x** (Standard, не Mono).
2. Импортировать папку проекта (`project.godot`) в редакторе.
3. Запустить главную сцену — `scenes/Main.tscn` (F5 в редакторе).

## Веб-экспорт

В Project Settings → Rendering → Renderer выставлен `gl_compatibility`
(обязательно для Яндекс Игр и слабых устройств).
В шаблонах экспорта выбирай **Web** — Godot 4.x + GL Compatibility
даёт работоспособный билд для встраивания в Яндекс Игры.

## Структура

```
scripts/
├── core/             # autoload-синглтоны
│   ├── EventBus.gd       — глобальные сигналы
│   ├── GameState.gd      — мета-прогресс между забегами
│   ├── RunState.gd       — состояние текущего забега (HP, золото, моды)
│   ├── SaveSystem.gd     — сейв/загрузка (Yandex SDK или локально)
│   └── YandexSDK.gd      — обёртка над Яндекс SDK (no-op в редакторе)
├── data/             # data-классы (Resource-like)
│   ├── TileType.gd       — типы тайлов и правила связывания
│   └── ChainResult.gd    — результат цепочки
└── battle/           # игровой бой
    ├── BoardLogic.gd     — чистая логика поля (без нод)
    ├── ChainResolver.gd  — превращает цепь в эффекты
    ├── Tile.gd           — визуал клетки
    ├── Board.gd          — визуальное поле
    ├── SwipeController.gd — обработка ввода
    └── Battle.gd         — корневая нода боя

scenes/
├── Main.tscn
└── battle/
    ├── Battle.tscn
    ├── Board.tscn
    └── Tile.tscn
```

## Архитектура — ключевые принципы

1. **Логика отделена от визуала.** `BoardLogic` и `ChainResolver` —
   обычные `Reference`-классы без нод. Их можно дёргать из юнит-тестов.
2. **Общение через EventBus.** Никто не знает о других подсистемах напрямую.
3. **Data-driven.** Враги, апгрейды, классы — будут лежать в `.tres`-файлах
   (`scripts/data/*` → пока минимум, расширяем по мере необходимости).
4. **YandexSDK — заглушка вне веб-сборки.** Игра одинаково работает в редакторе
   и на платформе.

## Что уже работает (MVP логики)

- Поле 6×6, генерация тайлов с шансом спавна врага.
- Свайп по соседним клеткам (8 направлений), включая возврат.
- Линковка одинаковых типов + спец-правило: меч ↔ враг.
- Цепочка считается по длине → даёт урон / щит / хил / золото.
- Множители комбо (GREAT / INSANE / GODLIKE), шанс крита, синергия меч+монета.
- Враги тикают после хода, бьют по таймеру.
- Гравитация и пополнение поля после расхода тайлов.
